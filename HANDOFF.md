## C'est quoi

`npx token-flow-manager <dossier>` — gestionnaire local de Design Tokens (DTCG 2025.10). Démarre un serveur Fastify local + un dashboard Angular « façon panneau Variables de Figma » pour visualiser/éditer les fichiers `.json` de tokens (édition bidirectionnelle, écriture atomique, format préservé).

### Architecture drag & drop (front)
`GroupDropRegistry` (dans `features/sidebar/`) est un **registre partagé** de TOUS les drop-lists (niveaux sidebar `gt-*`, zones « into » par dossier `gt-into-*`, sections du tableau `sec-*`) + une cible active (`activeTarget`) pour l'indicateur visuel. Tout est connecté à tout (`connectedTo = registry.ids()`).

**Routage par type via `cdkDropListEnterPredicate` (clé)** : les deux gestes sont séparés par le type du payload pour que CDK route correctement et qu'ils ne se gênent pas.
- listes de niveau `gt-*` → acceptent **uniquement les groupes** (`acceptsGroup`) : reorder des frères (ligne d'insertion) + re-nest cross-niveaux.
- zones « into » `gt-into-*` (par dossier, `sortingDisabled`) + sections tableau `sec-*` → acceptent **uniquement les tokens** (`acceptsToken`) : drop d'une variable dans un dossier (surlignage, **pas** de ligne noire).
- Pièges résolus : (a) sans ce filtre, les zones « into » qui recouvrent les lignes interceptaient le drag de reorder des groupes → reorder cassé ; le predicate fait que CDK les ignore pour un groupe et retombe sur la liste de niveau. (b) Le placeholder du tableau **doit** faire la hauteur exacte d'une ligne (**41px**) sinon la géométrie de swap CDK dérive en drag-bas (« il faut descendre 2 crans trop bas »).

La sélection multiple (tokens + groupes) vit dans `ProjectStore` (`selectedIds`/`selectedGroupKeys`), partagée table↔sidebar. Réactivité : `updateValue`/`reorder` = **updates optimistes** locaux puis réconciliation ; `refresh()` **coalescé** (un mutation émet 3 events WS → 1 seul refetch). `examples/multimode/` = fixture mode-repliée (preview pointe dessus via `.claude/launch.json`).
⚠️ **Le drag CDK ne se déclenche PAS via des PointerEvent synthétiques (eval)** — vérifié : aucun preview/placeholder ne se crée. Tester les gestes **à la souris**. Le backend (reorder/move/rename) est lui testable via l'API HTTP et l'est.

### Alias namespace de collection (RÉSOLU — gros impact)
Convention Tokens Studio/PrimeNG : un alias `{primitive.green.500}` nomme la **collection** en 1er segment (le token vit à `green.500` dans la collection `primitives`). Le resolver (`core/resolver.ts`) construit `collectionNamespaces` (nom + chaque segment `/` + variante singulier/pluriel) et, quand le chemin littéral n'existe pas, retombe sur la collection nommée. Vérifié sur les données réelles d'un projet de test : **534 → 0** broken-alias, que le projet soit ouvert à la racine (manifest) ou sur un sous-dossier de tokens (auto-détection).

## Conventions & pièges (IMPORTANT)

- **Édits avec littéraux `' '` (quote-espace)** : l'outil Edit a corrompu plusieurs fois des littéraux quote-espace (insertion de bytes de contrôle / multi-ligne). Éviter `x.join(' ')` ; préférer `JSON.stringify`/séparateurs nommés. Si un `grep` dit « binary file », c'est ça → vérifier avec `od -c`.
- **Clés JSON numériques** : `"50"`/`"900"` sont toujours sérialisées en ordre croissant par JS/JSON → pas de reorder possible (et le round-trip les normalise déjà). Le reorder ne marche que sur clés non-numériques.
- **Watcher** : NE PAS surveiller la racine récursivement (EMFILE). On surveille seulement `dirname` des fichiers de tokens (`startWatching`, depth 0).
- **Typage `type`** : c'est un `string` libre (pas l'enum DTCG) côté modèle ; `DtcgType` reste l'ensemble canonique. UI gère les types inconnus (glyphe ◇, pas de swatch).
- **Config priorité** : tokenflow.config.json hand-authored > manifest > auto. Un config `autoGenerated:true` ne fige PAS les collections (redétectées à chaque load). Un vieux config non-marqué bloque le manifest → le supprimer.
- **Preview réutilise l'onglet** : l'état signals peut « survivre » entre `preview_start` → recharger (`location.reload()`) avant de tester. Le drag CDK ne se simule pas fiablement en eval (events pointer) — tester à la souris.
- **Tests** : core/server via vitest avec `resolve.alias` `@tokenflow/*` → `src` (pas besoin de build). Web n'a pas de tests unitaires (vérif via preview).
- **Édition multi-thèmes (modes-fichier)** : éditer une valeur par thème écrit dans le 1er fichier (le suivi du fichier source par mode n'est pas fait). À corriger avec l'éditeur de manifest.
